### 응용 서비스 명
```
FireScene TwinLink
```

### 팀명
```
Twinkle (트윈클)
```

### 응용 서비스 개요
응용 서비스를 개발하게 된 배경, 내용 및 용도, IoT 기술 활용 방법, 타겟 및 수요계층 등에 대해 서술
```
최근 전기차 배터리 과충전으로 인한 화재 사고 보도가 잦아지면서, 실제 현장을 상시 통제하기 어려운 상황에서도 선제적으로 탐지·대응할 수 있는 방법이 필요하다고 판단했습니다. 이에 디지털 트윈 기반 시뮬레이션과 로봇 원격 출동을 결합한 “FireScene TwinLink” 서비스를 기획, 개발했습니다.

세종대학교 전역을 디지털 트윈으로 구현한 Meta Sejong 상에서 서비스가 동작합니다. 서비스의 목적은 실세계와 유사한 가상 캠퍼스에서 화재 징후를 조기에 탐지하고, 실제 로봇을 현장으로 파견해 상황을 인식, 전달함으로써 초기 대응 의사결정을 지원하는 것입니다.

서비스의 시나리오는 아래와 같습니다.
Isaac Sim에서 캠퍼스 내 여러 지점에 가상 센서 노드를 배치하고 온도·습도 등 환경 데이터를 지속 생성

센서 스트림에서 이상치 발생 시, 해당 지점의 좌표를 식별

사용자가 로봇 출동 명령을 발행하면 로봇이 지정 좌표로 자율 주행(SLAM) 이동

로봇이 현장 이미지를 촬영·전송하여 화재 의심 구역의 시각 정보를 제공

관리자는 화면에서 상황을 확인하고 추가 대응(경보, 대피 유도, 진화 지원 등)을 결정

화재는 언제 어디서든 발생할 수 있습니다. 저희 FireScene TwinLink 서비스는 시간과 장소에 구애받지 않고 화재 징후가 탐지되는 즉시, 디지털 트윈과 실제 로봇을 연동한 대응 체계를 가동하여 현장을 신속히 확인하고, 의사결정에 필요한 정밀 정보를 제공함으로써 초기 대응의 골든타임을 확보합니다. 나아가 캠퍼스, 산업현장, 도시 관제까지 확장 가능한 실증 플랫폼으로서, 안전 운영의 표준화를 목표로 합니다.
```

### 응용 서비스 구현 방법
응용 서비스 구현을 위한 디바이스 제작 방법, 플랫폼 연동을 통한 응용서비스 구현 방법, IoT 기술 활용 방법 등에 대해 서술
```
데이터 수집은 Isaac Sim에서 발생한 센서 데이터를 1초 주기로 MQTT를 통해 oneM2M 표준 플랫폼(Mobius CSE)에 cin 리소스로 적재하는 방식으로 진행됩니다. 이상 탐지는 사용자가 Mobius에 저장된 시계열을 구독·가공하여 임계치/통계 기반 이상치를 검출하게 됩니다. 로봇에게 이동 명령을 전송하는 방안은 로봇 출동은 Mobius의 명령 컨테이너에 cin을 생성하는 방식으로 발행하며, ROS 측에서 해당 컨테이너를 subscribe하여 액션을 실행하게 됩니다. 로봇은 SLAM으로 목적 좌표까지 이동 후 현장 이미지를 촬영해 Mobius에 업로드, 운영 화면은 이를 실시간 표시하게 됩니다.

사용되는 핵심 인터페이스는 MQTT Pub/Sub, oneM2M 리소스 모델(cnt, cin), ROS 액션/토픽, SLAM 내비게이션 스택입니다.
```

### 참가팀의 개발 역량
응용 서비스 개발을 위한 참가팀의 역량, 개발 경험 등 참가팀의 전문성 및 개발 역량을 서술
```
저희 팀은 세종대학교 자율지능시스템 연구실에서 로봇 지능과 디지털 트윈 응용을 연구, 개발해 온 석사 1명, 학사 2명으로 구성되어 있습니다. 연구실 과제와 교내외 프로젝트를 통해 이어지는 로봇 대응 파이프라인을 직접 설계, 구현해 왔으며, 연구와 실서비스 개발을 병행하는 과정에서 문제 정의, 실험 설계, 빠른 프로토타이핑과 검증 사이클을 체득했습니다. 운영체제는 macOS와 Linux를 주로 사용하고, 개발 언어는 Python을 중심으로 하며, ROS2와 NVIDIA Isaac Sim을 활용한 로봇 제어, 시뮬레이션에 숙련되어 있습니다.

이 경험은 이번 응용 서비스 개발에 직접 연결됩니다. 실제 화재 현장을 사용하지 않고도 Meta Sejong 상에서 다양한 위험 시나리오를 반복적으로 검증할 수 있으며, 필요 시 센서 종류와 로봇 플랫폼을 확장해 복수 지점을 동시 모니터링하는 구조로 확장할 수 있습니다. 요약하면, 저희 팀은 디지털 트윈–IoT–로봇을 표준 프로토콜과 시뮬레이션 도구로 유기적으로 묶어, “탐지-출동-현장 인식-의사결정”의 폐쇄 루프를 빠르고 신뢰성 있게 구현, 운영할 수 있는 실전형 개발 역량을 갖추고 있습니다.
```

### 응용서비스 기대효과
응용 서비스를 통해 나타날 수 있는 사회적, 경제적 파급 효과에 대해 간략하게 서술
```
FireScene TwinLink는 화재 징후를 조기에 포착하고 로봇을 즉시 출동시켜 현장을 영상으로 확인함으로써 골든타임 내 대응을 가능하게 합니다. 실제 화재를 유발하지 않고도 캠퍼스나 산업현장에서 고위험 시나리오를 반복적으로 훈련할 수 있어, 안전 매뉴얼과 관제 체계가 점차 정교해지고 야간, 무인 시간대에도 상시 감시가 이어집니다. 더불어 센서–관제–로봇을 MQTT와 oneM2M, ROS 같은 표준으로 연결하기 때문에 이기종 시스템 간 호환성이 개선되고, 수집 및 분석된 로그를 바탕으로 정책을 점검·개선하는 데이터 중심의 안전 문화가 자리잡습니다. 결과적으로 지역사회 전반의 체감 안전도가 높아지고, 인명 피해와 2차 피해(연기 확산, 폭발 등)를 유의미하게 줄일 수 있습니다.

경제적으로는 초기 대응의 정확도가 높아짐에 따라 설비, 재고 손실과 가동 중단 같은 직간접 피해가 감소합니다. 위험 지역을 사람 대신 로봇이 선행 정찰하므로 불필요한 인력, 차량 출동이 줄고, 산업재해 보험 비용 부담도 완화됩니다. 또한 시뮬레이터 기반 검증으로 PoC에서 상용화까지의 기간과 비용을 절감할 수 있으며, 기존 센서 네트워크 관제 인프라와 모듈형으로 통합되기 때문에 단계적 도입이 가능해 총소유비용을 낮춥니다. 축적된 레퍼런스를 기반으로 스마트캠퍼스, 스마트시티, 산업단지 안전관제 시장으로 확장할 수 있고, 표준 기반 구조를 강점으로 공공, 민간 영역에 솔루션 수출 가능성도 확보하게 됩니다.
```
